Reflective Cellular Automata

Note: this is a work in progress. Things may be broken, incomplete, or change without notice.

Elementary Cellular Automata (ECA) assign each cell a binary state updated by a fixed 3-cell neighborhood rule. There are 256 such rules; see Wikipedia or MathWorld for a full survey.

Fuzzy automata extend ECA by allowing continuous values on [0, 1] instead of strict binary states, producing smoother, gradient-like dynamics with the same neighborhood logic.

This simulator (Reflective Cellular Automata) goes further: the grid encodes both state and its own transition rules (there is no separate rule table). Each cell value serves simultaneously as a state and as the output weight for a neighborhood pattern, so the automaton rewrites itself as it evolves.

Concretely: the grid has 8 cells, each holding a real value and simultaneously serving as the output weight for one of the 8 binary neighborhood patterns (000 to 111). On each step, a cell's new value is derived from its local neighborhood via the selected fuzzy match function and topology.

The topology selector controls how the 8 cells are connected as spatial neighbors. The cells correspond to the 8 vertices of a 3-bit hypercube (Q3), so adjacency is defined by which vertices share a cube edge.

  • Ring 1–6: cells are arranged in a 1D ring following one of six distinct Hamiltonian cycles through Q3; each cell has two neighbors, both differing by exactly one bit, and the ring position determines the spatial left-right order.
  • Q3: uses the full hypercube structure; each cell's neighborhood consists of its three bit-adjacent cells (those differing by exactly one bit), not including the cell itself. This adjacency is uniquely defined by the cube with no arbitrary choice of ordering. Cell positions in the visualization carry no spatial meaning, unlike the ring topologies where position reflects the ring order.
  • Dynamic: each step, cells are reassigned to ring positions so that each cell's immediate neighborhood resembles the pattern that cell encodes, maximizing total alignment across all cells.

The fuzzy match function measures how well a cell's neighborhood fits each binary pattern. Selected with the Fuzzy selector.

  • Łukasiewicz: max(0, a+b+c−2), the standard t-norm; the sum must exceed 2 with a hard zero floor.
  • Interference: (√a+√b+√c−2)², squaring a sum of amplitudes to produce wave-like cancellation.
  • Product: a·b·c, strict conjunction requiring all three inputs to be simultaneously high.

Conservation mode controls how the total activation is preserved across steps.

  • No conserve: the standard unconstrained fuzzy ECA update; total activation drifts freely.
  • Renorm: the next state is computed normally, then rescaled globally so its sum matches the previous step. A post-hoc correction, not a structural constraint.
  • Spatial flux: each cell's value is treated as conserved mass flowing between neighboring positions, distributed to adjacent positions (left/self/right for ring topologies; three bit-adjacent cells for Q3) in proportion to their unconstrained next-values. Conservation is exact and local.
  • Pattern flux: as in ECA, each pattern's rule value drives the update; but instead of being summed directly into cells, it is treated as mass and distributed to cells in proportion to how well their neighborhoods match that pattern, normalized so each pattern's total contribution is conserved.
  • Pattern flux (inv): each cell distributes its mass to pattern entries in proportion to neighborhood match, normalized over all patterns.
