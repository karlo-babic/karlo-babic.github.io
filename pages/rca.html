Reflective Cellular Automata

Fuzzy:

Elementary Cellular Automata (ECA) assign each cell a binary state updated by a fixed 3-cell neighborhood rule. There are 256 such rules; see Wikipedia or MathWorld for a full survey.

Fuzzy automata extend ECA by allowing continuous values on [0, 1] instead of strict binary states, producing smoother, gradient-like dynamics with the same neighborhood logic.

This simulator (Reflective Cellular Automata) goes further: the grid encodes both state and its own transition rules (there is no separate rule table). Each cell value serves simultaneously as a state and as the output weight for a neighborhood pattern, so the automaton rewrites itself as it evolves.

Concretely: the grid has 8 cells in a ring, each holding a value in [0, 1]. On each step, a cell's new value is computed as a weighted sum over all 8 possible binary neighborhood patterns (000 to 111). For each pattern, a fuzzy match function measures how well the actual left-center-right values fit that pattern. The match score is multiplied by the cell whose index corresponds to that pattern, and the 8 products are summed and clamped to [0, 1]. The pattern-to-cell mapping uses a Gray code ordering so adjacent patterns differ by exactly one bit.

The fuzzy match can be switched using the Fuzzy selector. Sharp Lukasiewicz gives a smooth square-root blend. Nilpotent and Dual Nilpotent apply a hard threshold before taking the minimum. Interference squares a sum of square roots, which produces amplitude-like cancellation effects.

When energy conservation is on, the total sum of all cell values is rescaled after each step to match the previous total, which keeps the overall activation from drifting to zero or saturating to one.
